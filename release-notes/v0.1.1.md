## Fixed

- **Status window no longer freezes on "closing"** when the tunnel drops on its own. Previously the disconnect button was disabled and labelled `closing` while the main process was still waiting for a click that could never come — a deadlock. The button now stays clickable (`close`) so the user can dismiss the screen.

## Added

- **Auto-reconnect with backoff.** When `openconnect` exits unexpectedly (TLS drop, network blip, NAT timeout, etc.) the app now calls `gateway.doLogin()` to obtain a fresh auth cookie and re-spawns `openconnect`. Backoff schedule is `2s → 5s → 10s → 20s → 30s → 60s` (capped). Session uptime keeps running across attempts. The status window shows a countdown and the attempt number, e.g. `reconnecting in 5s (attempt 2)`.
- **Disconnect returns to the gateway selector instead of quitting.** Pressing Disconnect now closes the tunnel and reopens the gateway/region picker, so you can switch regions (or reconnect to the same one) without re-running SAML. To exit the app, use the tray menu's _Quit gpsaml_ or close the gateway selector window.
- **Reconnect attempts are cancellable.** During the backoff countdown the button is labelled `cancel`; pressing it aborts the retry loop and returns to the gateway selector.

## Known limitations

- If the portal's `portal-userauthcookie` itself expires (typically next day or later), reconnect cannot recover and the status window shows `session expired — please reconnect`. You currently need to quit and relaunch the app to re-run SAML; a future release will handle this in-place.
- The underlying TLS drop cannot be prevented from the client side (macOS sleep, Wi-Fi roam, NAT idle timeout, etc.) — this release only ensures the app reconnects cleanly afterwards.
