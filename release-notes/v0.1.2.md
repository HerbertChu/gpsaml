## Fixed

- **UI no longer freezes during long sessions.** `openconnect` was being launched with `--dump-http-traffic`, which streamed every HIP/keepalive HTTP exchange into the main process — Electron's event loop got starved over hours/days of uptime, leaving the renderer unresponsive (the tunnel itself kept running in its own child process). The flag is removed; routine connections now produce only modest log output.
- **Reconnect listener leak.** The `sleep` and `waitClose` helpers inside the auto-reconnect loop added `'abort'` listeners to the shared `AbortSignal` that were only cleaned up on cancellation, so frequent reconnects piled them up (and pinned dead `ChildProcess` references). Both helpers now remove their listeners on the happy path.
- **Tray "Disconnect" reflects current state.** The menu item was always enabled, even on the gateway-selection screen where there was nothing to disconnect. It now enables/disables based on whether a VPN child process is actually alive.
- **Tray "Disconnect" actually disconnects.** It used to just `SIGTERM` the child, which the reconnect loop interpreted as an unexpected tunnel drop and immediately reconnected. It now routes through the same path as the in-window Disconnect button, aborting the loop and showing the proper "tearing down" UI.
- **Enter on the gateway list now connects.** The form had no input element, so Enter was being swallowed; only `↑`/`↓` were handled. Enter now submits the selection (matching the on-screen hint).
- **"scanning portal" no longer sticks around after the list loads.** The class-level `display: flex` rule on `.loading` outranked the UA default for `[hidden]`, leaving the spinner visible underneath the populated list. A `[hidden] { display: none !important; }` rule restores the expected behavior.
- **Copy / paste works in the portal address field.** The chrome-wide `user-select: none` was leaking into the `<input>`, breaking Cmd+A and paste. The field now re-enables text selection locally.

## Added

- **"Remember this address" on the portal screen.** Tick the box before pressing Authenticate and the address is saved to user data; next launch pre-fills it. Untick (or clear the field) to forget. The address is only saved after a successful prelogin, so typos aren't remembered.
- **Orphan openconnect detection at startup.** When gpsaml is force-killed (UI freeze, sudo prompt timeout, etc.) its `openconnect` child gets reparented to launchd and continues holding the utun interface up and the default route hostage — the next portal connection then fails with `EADDRNOTAVAIL`. On launch the app now scans for these leftovers and prompts before continuing: _Terminate and continue_, _Continue anyway_ (for users intentionally running another openconnect), or _Quit gpsaml_.
- **Bounded log file.** `/tmp/gpsaml.log` is now capped at 20 MB; on rollover the previous file is renamed to `/tmp/gpsaml.log.1` (one archive kept).
- **`GPSAML_LOG_LEVEL` env var.** Defaults to `info`; set to `debug`/`trace` when reporting issues. Propagated through the sudo relaunch.
